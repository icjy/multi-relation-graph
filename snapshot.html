特征说明

策略特征宽表中新加工字段的计算口径。

通用口径

以下说明以宽表当前行为准：U 为当前行 app_user_id，A 为当前行 consigneeMobileId，C 为 addr_cluster_id，D 为 device_id，P 为 ip。

  • 申请笔数类字段统计筛选后的全量申请行。
  • 贷款笔数和坏账率类字段只统计 final_result=30 的通过样本。
  • 坏账率 = 通过样本中逾期笔数 / 通过样本总笔数。
  • 所有坏账率类字段的逾期判断都跟随页面“逾期口径”下拉框，例如选择 fpd1 就按 fpd1 判断逾期，选择 cpd7 就按 cpd7 判断逾期。
  • 关联人数、关联设备数、关联 IP 数、关联地址簇数均为去重计数。

实体维度

收货人手机号关联人数/设备数/ip数/地址簇数
A 关联过的用户、设备、IP、地址簇去重数量。
收货人手机号坏账率
A 关联记录中通过且逾期的笔数 / A 关联记录中通过笔数。
地址簇相关字段
以 C 聚合，统计去重地址、用户、中介、设备、IP、申请笔数和坏账率。
设备相关字段
以 D 聚合，统计去重用户、地址簇、中介、申请笔数和坏账率。
IP相关字段
以 P 聚合，统计去重用户、设备、申请笔数和坏账率。
借款人一度中介数/贷款笔数/坏账率
以 U 聚合，统计直接中介数、通过贷款笔数和通过样本坏账率。

二度关系

二度借款人数
U 的一度中介关联过的其他 app_user_id 去重数，去掉 U 自己。
二度中介数
A 的一度借款人关联过的其他 consigneeMobileId 去重数，去掉 A 自己。
二度设备数/ip数/地址簇数
U 的二度借款人集合下所有记录中的设备、IP、地址簇去重数。
二度申请人数/申请笔数
U 的二度借款人集合中有申请记录的去重人数，以及对应全量申请行数。
二度贷款笔数/坏账率
U 的二度借款人集合中 final_result=30 的笔数，以及其中逾期笔数占比。

共享用户中介投影

两个中介共享同一个 app_user_id 时连边，边权就是这两个中介共享的去重借款人数。

节点度数(共享用户)
A 直接连接的其他中介数量。
加权度数(共享用户)
A 连到的每一条边权相加，边权 = A 与对应中介共同关联的去重 app_user_id 数。
PageRank分数(共享用户)
A 在共享用户中介投影图上的 PageRank。
中介中心性(共享用户)
A 在共享用户中介投影图上的介数中心性：BC(A)=Σσ(s,t|A)/σ(s,t)，再除以 (n-1)*(n-2) 归一化；衡量 A 是否位于其他中介之间的最短路径上。
连通子图大小/社区id(共享用户)
A 所在共享用户连通分量的中介数量和 ID。

连通分量关系图

两个中介通过共享用户、设备、IP、地址簇计算关系；设备/IP/地址簇比较两个中介各自名下直接申请记录的属性集合交集，不要求必须共享借款人。

仅首贷或复贷
共享Device >= 1，或共享User >= 1，或共享IP >= 2 且 IP人数 <= 20，或共享Addr >= 1。
全部数据
共享Device >= 1，或共享User >= 2，或共享IP >= 2 且 IP人数 <= 50，或共享Addr >= 1。
节点度数(连通分量关系)
A 在过滤后关系图上直接连接的其他中介数量。
加权度数(连通分量关系)
A 连到的每一条边权相加，边权 = 共享User*1.0 + 共享Device*1.2 + 共享IP*0.8 + 共享Addr*1.0。
PageRank分数/中介中心性(连通分量关系)
A 在过滤后关系图上的 PageRank 和介数中心性。
连通子图大小/社区id(连通分量关系)
A 所在过滤后中介连通分量的规模和 ID。

多关系中介投影

多关系边权 = 1.0 * 共享用户数 + 1.2 * 共享设备数 + 0.8 * 共享 IP 数 + 1.0 * 共享地址簇数；设备/IP/地址簇使用中介名下直接申请记录口径。

节点度数(多关系)
A 在多关系图上直接连接的其他中介数量。
加权度数(多关系)
A 连到的每一条边权相加，边权 = 1.0 * 共享用户数 + 1.2 * 共享设备数 + 0.8 * 共享 IP 数 + 1.0 * 共享地址簇数。
PageRank分数(多关系)
A 在多关系图上的 PageRank。
中介中心性(多关系)
A 在多关系图上的介数中心性。
连通子图大小/社区id(多关系)
A 所在多关系连通分量的中介数量和 ID。

边权与 PageRank

Agent 多关系边权
1.0 * 共享用户数 + 1.2 * 共享设备数 + 0.8 * 共享 IP 数 + 1.0 * 共享地址簇数；共享设备/IP/地址簇来自两个中介各自名下直接申请记录的属性集合交集。
User 同构图边权
1.0 * 共享中介数 + 1.2 * 共享设备数 + 0.8 * 共享 IP 数 + 1.0 * 共享地址簇数。
PageRank 公式
PR(v) = (1-d)/N + d*Dangling/N + d*Σ PR(u)*w(u,v)/Σw(u,x)，d=0.85，迭代 40 轮。
符号说明
N 为节点数，w 为边权，Dangling 为无出边节点 PageRank 之和。

社区特征

连通分量时社区字段带 (Agent) 后缀，表示过滤后中介关系图社区；Louvain / Leiden 时社区字段带 (User) 后缀，表示用户同构图社区，并额外生成未过滤 Agent-Agent 多关系图社区字段。

...(Agent)
中介社区字段。连通分量默认输出这组字段；选择 Louvain / Leiden 时，也会额外输出基于未过滤 Agent-Agent 多关系图的这组字段。
...(User)
用户社区字段。选择 Louvain / Leiden 时输出，来自用户同构图社区。
社区规模
(Agent) 为社区中介数；(User) 为用户社区借款人数。
社区借款人数/贷款笔数/坏账率
社区覆盖用户数、通过贷款笔数和通过样本坏账率。
社区设备/ip/地址密度
社区内部节点两两共享对应属性形成的边数 / 理论最大边数。
社区共享借款人数
(Agent) 为社区内被至少两个中介共享的借款人数；(User) 当前为 0。
社区风险分
由坏账率、网络密度、设备/IP/地址密度等组合得到的解释性排序分。
Agent 风险分
min(100, round(坏账率*55 + 网络密度*20 + min(共享借款人占比,1)*15 + min(中介数/10,1)*10, 2))。
User 风险分
min(100, round(坏账率*55 + 用户网络密度*15 + 设备密度*10 + IP密度*10 + 地址密度*10, 2))。
